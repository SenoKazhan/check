# Warehouse CV

Автоматизированное определение габаритов товаров по изображениям и их оптимизация размещения в упаковке.

Проект реализует программную альтернативу промышленным 3D-сканерам, используя методы монокулярной оценки глубины и алгоритмы целочисленного программирования для решения задачи трёхмерной упаковки (3D Bin Packing Problem).

## Ключевые возможности

* **Монокулярная оценка глубины:** Инференс модели Depth Anything V2 в среде выполнения OpenVINO для построения карты глубины по стандартным 2D-изображениям.
* **Метрическая калибровка:** Переход от относительной глубины к абсолютным значениям (мм) с использованием маркеров ArUco в качестве эталона масштаба.
* **3D-упаковка (CP-SAT):** Генерация вариантов размещения товаров в контейнере с учётом ограничений непересечения и гравитации на базе Google OR-Tools.
* **Асинхронная обработка:** Разделение веб-сервера и ресурсоёмких вычислений (CV, решатель) через очередь задач Celery и брокер Redis.
* **Интерактивная 3D-визуализация:** Рендеринг схемы укладки на стороне клиента с использованием Three.js.
* **Безопасность и RBAC:** JWT-аутентификация, разграничение ролей, защита от брутфорса (Rate Limiting).

## Архитектура и проектирование

Система спроектирована с соблюдением принципов Clean Architecture и SOLID:
* **Слой представления (API):** Тонкие контроллеры FastAPI, использующие внедрение зависимостей (`Depends`) для изоляции от бизнес-логики.
* **Слой бизнес-логики (Services/Domain):** Оркестраторы рабочих процессов, верификаторы и доменные исключения.
* **Слой доступа к данным (Repositories):** Инкапсуляция SQL-запросов, защита от инъекций через параметризованные запросы SQLAlchemy.
* **CV-конвейер:** Вынесен в изолированные Celery-воркеры для предотвращения блокировки event-loop'а веб-сервера.

## Быстрый старт

### Системные требования
* ОЗУ: минимум 8 ГБ (инференс модели на CPU требует значительных ресурсов).
* ПО: Docker Engine 24.0+, Docker Compose v2+.

### Развёртывание
```bash
git clone https://github.com/your-username/warehouse-cv.git
cd warehouse-cv
cp .env.example .env
docker-compose up -d --build
```
*При первом запуске контейнер `backend` автоматически применит миграции Alembic и инициализирует базу данных.*

### Доступ к интерфейсам
* **Frontend:** http://localhost:3000
* **Backend (Swagger UI):** http://localhost:8001/docs
* **Health Check:** http://localhost:8001/health

### Учётные данные по умолчанию
* **Логин:** `admin@warehouse.dev`
* **Пароль:** `admin123`

## Требования к входным данным

Для корректной работы модуля компьютерного зрения необходимо:
1. Наличие в кадре ArUco-маркера известного физического размера (в настройках можно задать значение).
2. Загрузка трех ортогональных ракурсов: Front (спереди), Side (сбоку), Top (сверху).
3. Маркер и товар должны находиться в одной физической плоскости.
4. Формат изображений: JPEG/PNG, разрешение не менее 640x480.
